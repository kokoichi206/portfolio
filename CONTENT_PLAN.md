## Data Management & Update Strategy

### Structure

```
/data
  profile.json        # hero, about
  skills.json         # all skills with categories
  certifications.json # cert list
  experience.json     # work history
  projects.json       # tier 1-3 projects
  writing.json        # featured articles
```

### Update Flow

1. Edit JSON files
2. git push to main
3. CI/CD auto-deploy (Vercel or Cloudflare Pages)

### Auto-Updates

- Blog: RSS feed via ISR (revalidate every 24h)
- GitHub contributions: GitHub API dynamic fetch
- Everything else: manual JSON edit + push

### When to Update

- New project/OSS: add to projects.json
- New job/role: add to experience.json
- New certification: add to certifications.json
- Blog: automatic via RSS
- Skills: update when significant new experience gained
